# RFC 04: Type Safety (Eliminate `any` with Type Guards)

## Problem Statement

Currently, Payload CMS defines the authenticated `user` object using a generic union type that lacks the specific custom fields present on our target collections (e.g., the `hasCompletedTour` boolean on `Members`, or checking the `roles` array on an admin `User`).
To bypass compilation errors, the codebase forcefully casts the union string as `any` (e.g., `(user as any).hasCompletedTour`).
This completely defeats the purpose of TypeScript, exposing the application to runtime exceptions if schema properties change without updating the frontend.

## Proposed Solution: Global Type Guards

We will eliminate the `any` keyword by introducing strongly-typed Custom Type Guards. By creating `isMember` and `isAdminUser` utilities, we allow the TypeScript compiler to automatically infer exact fields for the remainder of a code block.

---

## Technical Implementation Details

### 1. New Utility File (`src/lib/auth/typeGuards.ts`)

We will create a centralized utility file that imports the exact Types generated by Payload's `generate:types` script.

```typescript
import type { User, Member } from '@/payload/payload-types'

/**
 * Type guard to safely narrow the generic user object to the Member collection type.
 */
export function isMember(user: any): user is Member {
  return user && typeof user === 'object' && user.collection === 'members'
}

/**
 * Type guard to safely narrow the generic user object to the Admin User collection type.
 */
export function isAdminUser(user: any): user is User {
  return user && typeof user === 'object' && user.collection === 'users'
}
```

### 2. Code Refactor Example (Before & After)

**Before (e.g., in `src/app/app/page.tsx`)**:

```typescript
const initialData = {
  // ...
  hasCompletedTour: (user as any).hasCompletedTour || false,
}
```

**After**:

```typescript
import { isMember } from '@/lib/auth/typeGuards'

let hasCompletedTour = false
if (isMember(user)) {
  hasCompletedTour = user.hasCompletedTour || false
}

const initialData = {
  // ...
  hasCompletedTour,
}
```

### 3. Files Identified for Refactoring

A codebase scan using `grep_search` for `user as any` revealed the following files that currently violate type safety and will be refactored:

1. `src/app/app/page.tsx`
2. `src/payload/utils/access/access.ts`
3. `src/payload/utils/access/accessField.ts`
4. `src/payload/utils/visibleFor.ts`
5. `src/payload/collections/settings/Users/hooks/ensureRolesPopulated.ts`

For each of these files, we will import the necessary Type Guards and wrap the `any` casting logic in secure `if()` blocks.
